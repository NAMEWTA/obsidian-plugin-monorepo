# Command 模板

完整的 command 文件模板，复制后根据实际需求填充。

```markdown
---
command: <command-name>
description: <一句话描述命令的核心功能>
tags:
  - <tag1>
  - <tag2>
---

# <Command 标题>

## 用途

<描述命令的核心目的、解决什么问题、适用场景>

## 指令内容

你是一个专业的 <领域> 助手，负责 <核心职责>。

## 输入参数

- **<参数名1>**（必填）：<参数说明>，如 `<示例值>`
- **<参数名2>**（可选）：<参数说明>，默认值为 `<默认值>`

## 任务

### 1. <步骤名称>

<步骤说明>

```bash
<具体命令>
```

### 2. <步骤名称>

<步骤说明>

| 条件 | 处理方式 |
|------|----------|
| <条件1> | <处理1> |
| <条件2> | <处理2> |

### 3. <步骤名称>

<步骤说明>

## 约束

- <约束条件1>
- <约束条件2>
- 遵循 [<外部规范>](<规范链接>) 标准

## 输出格式

```
📋 <摘要标题>
-----------
<摘要内容>

📝 <详情标题>
-----------
- <详情项1>
- <详情项2>

✅ 操作结果
-----------
- [x] <已完成项1>
- [x] <已完成项2>
- [ ] <待完成项>
```

## 注意事项

- <注意事项1>
- <注意事项2>
- 如果遇到 <异常情况>，需要 <处理方式>
```

## 章节说明

### 用途
- 1-3 句话说明命令解决的问题
- 明确适用场景和前置条件

### 指令内容
- 定义 Claude 的角色（如"版本发布助手"）
- 简要说明核心职责

### 输入参数
- 使用粗体标注参数名
- 明确标注必填/可选
- 提供示例值帮助理解

### 任务
- 使用编号的三级标题组织步骤
- 每步原子化：一个步骤只做一件事
- 包含具体命令（bash 代码块）
- 复杂逻辑使用表格明确条件分支

### 约束
- 列出操作边界和限制
- 引用外部规范（如 Conventional Commits）
- 明确"只做什么"和"不做什么"

### 输出格式
- 使用 emoji 增强可读性
- 使用分隔线区分段落
- 使用 checklist 展示执行结果
- 使用占位符（`<占位符>`）标注动态内容

### 注意事项
- 列出异常处理策略
- 说明前置条件检查
- 提供回退方案
